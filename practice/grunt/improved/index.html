This is an example of how we can improve Grunt.

Two tasks available:

  • grunt build: To build files for production
  • grunt or grunt default: Starts grunt to watch file changes

Both tasks changes index.html file. So, after you run each task, checkout JS and CSS files in network tab to see the differences.

We are going to use the following plugins:

  • Load Grunt Tasks:
    • This plugin is responsible for loading tasks accordingly to package.json file, instead of doing loadNpmTasks for all your packages.
    • Link
  • Load Grunt Config:
    • This plugin is responsible for modularizing Grunt Tasks, ie, we can put each task file separately from Grunt Config File (Gruntfile.js).
    • Link
  • JSHint:
    • Checks Javascript syntax.
    • Link
  • Compass:
    • Compiles SASS files to CSS.
    • Link
  • Grunt Uglify:
    • Minify and Uglify Javascript files.
    • Link
  • Grunt watch:
    • Watch for file changes.
    • Link
  • Grunt Text Replace:
    • Replace texts in files.
    • Link
